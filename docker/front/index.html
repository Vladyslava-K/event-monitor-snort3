Snort - Open Source Intrusion Prevention System (IPS)

Snort is the foremost Open Source Intrusion Prevention System (IPS) in the world. Snort IPS uses a series of rules that help define malicious network activity and uses those rules to find packets that match against them and generates alerts for users.

Snort can be deployed inline to stop these packets as well. Snort has three primary uses: As a packet sniffer like tcpdump, as a packet logger â€” which is useful for network traffic debugging, or it can be used as a full-blown network intrusion prevention system.

Python Client-Server Monitor for Snort3

The application you are using now is a client-server monitor for Snort3 in Python. It receives Snort3 intrusion events, parses obtained data, stores it in a PostgreSQL database, and provides a REST API to the client to retrieve summary reports of the stored events.

App Capabilities:

  • Track and store events.
  • Provide alert counts for each alert SID observed during specified time period (last day/week/month/all recorded).
  • Provide alert counts for each unique client/server address during specified time period (last day/week/month/all recorded).
  • Provide recorded alerts for a given client/server address.
  • Clear alert records stored in the database.
  • Dump rules used by Snort to a file, write them into DB, and provide API for them with filters by gid, sid, and action.
  • New local rules can be added to file from the client.
  • Rules packages can be updated using pulledpork3.
  • Provide the log of all operations in a given time range (between two UTC timestamps; the range has to be less than a week).
  • Server is running in daemon mode.
  • Snort is running on an interface.
  • Snort is running with shell support. Users can run an arbitrary shell command and see the response.
  • Performance monitor is enabled, metrics are stored to MongoDB.
  • Rule profiling is enabled.

Performance Monitoring Requests:

  • Get peg count reports between two timestamps.
    • params:
    • begin (timestamp)
    • end (timestamp)
    • prefix (string, optional) - filter peg counts by name prefix
  • Get peg count delta between two records which are closest to the provided timestamps.
    • params:
    • begin (timestamp)
    • end (timestamp)
    • prefix (string, optional) - filter peg counts by name prefix

Rule Profiling Requests:

  • Start rule profiling for a given amount of time.
    • params:
    • time - time, in minutes, to run rule profiling for
    • until - timestamp after which the profiling should stop. Either time or until has to be provided, but not both.
  • Get last stored rule profiling result.
